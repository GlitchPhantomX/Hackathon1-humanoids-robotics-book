# डिप्लॉयमेंट गाइड: बहु-भाषा अनुवाद प्रणाली

## अवलोकन
यह गाइड भौतिक एआई रोबोटिक्स पाठ्यपुस्तक के लिए बहु-भाषा अनुवाद प्रणाली को डिप्लॉय करने के लिए चरण-दर-चरण निर्देश प्रदान करता है। प्रणाली में अनुवाद क्षमताओं के साथ एक डॉकुसॉरस फ्रंटएंड और प्रमाणीकरण और अनुवाद सेवाओं के लिए एक फास्टएपीआई बैकएंड शामिल है।

## पूर्वापेक्षाएँ

### सिस्टम आवश्यकताएँ
- **नोड.जेएस**: संस्करण 18 या उच्चतर
- **पायथन**: संस्करण 3.8 या उच्चतर
- **एनपीएम** या **यार्न** पैकेज प्रबंधक
- **गिट** संस्करण नियंत्रण के लिए
- **ओपनएआई एपीआई कुंजी** के लिए पहुंच (अनुवाद उत्पादन के लिए)

### पर्यावरण आवश्यकताएँ
- स्थैतिक फ़ाइलें सेवा करने में सक्षम वेब सर्वर (फ्रंटएंड के लिए)
- पायथन समर्थन के साथ बैकएंड सर्वर (फास्टएपीआई बैकएंड के लिए)
- डेटाबेस (पोस्टग्रेसक्यूएल अनुशंसित)
- सुरक्षित कनेक्शन के लिए एसएसएल प्रमाणपत्र

## बिल्ड निर्देश

### फ्रंटएंड (डॉकुसॉरस) बिल्ड

1. **फ्रंटएंड डायरेक्टरी में जाएं**:
   ```bash
   cd physical-ai-robotics-textbook/docusaurus
   ```

2. **निर्भरताएँ स्थापित करें**:
   ```bash
   npm install
   # या
   yarn install
   ```

3. **स्थैतिक साइट बनाएं**:
   ```bash
   npm run build
   # या
   yarn build
   ```

4. **बिल्ड की पुष्टि करें**:
   ```bash
   npm run serve
   # या
   yarn serve
   ```
   साइट `http://localhost:3000` पर उपलब्ध होगी

### बैकएंड (फास्टएपीआई) बिल्ड

1. **बैकएंड डायरेक्टरी में जाएं**:
   ```bash
   cd physical-ai-robotics-textbook/backend
   ```

2. **एक वर्चुअल वातावरण बनाएं** (अनुशंसित):
   ```bash
   python -m venv venv
   source venv/bin/activate  # विंडोज पर: venv\Scripts\activate
   ```

3. **पायथन निर्भरताएँ स्थापित करें**:
   ```bash
   pip install -r requirements.txt
   ```

4. **पर्यावरण चर सेट करें** (नीचे पर्यावरण चर अनुभाग देखें)

## पर्यावरण चर

### फ्रंटएंड (डॉकुसॉरस डायरेक्टरी में .env फ़ाइल)
```bash
# एपीआई एंडपॉइंट्स
REACT_APP_API_URL=http://your-backend-domain.com
REACT_APP_AUTH_API_URL=http://your-backend-domain.com

# अनुवाद उत्पादन के लिए ओपनएआई एपीआई कुंजी
REACT_APP_OPENAI_API_KEY=your_openai_api_key_here

# वैकल्पिक: कस्टम कॉन्फ़िगरेशन
REACT_APP_DEFAULT_LANGUAGE=en
REACT_APP_ENABLE_TRANSLATION_LOGGING=true
```

### बैकएंड (बैकएंड डायरेक्टरी में .env फ़ाइल)
```bash
# डेटाबेस कॉन्फ़िगरेशन
DATABASE_URL=postgresql://username:password@localhost:5432/your_database_name
NEON_DATABASE_URL=your_neon_database_url

# प्रमाणीकरण
SECRET_KEY=your_very_long_secret_key_here
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30

# ओपनएआई एपीआई
OPENAI_API_KEY=your_openai_api_key_here

# सीओआरएस के लिए फ्रंटएंड यूआरएल
FRONTEND_URL=http://your-frontend-domain.com

# वैकल्पिक: प्रमाणीकरण के लिए ईमेल कॉन्फ़िगरेशन
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your_email@gmail.com
SMTP_PASSWORD=your_app_password
```

## डिप्लॉयमेंट चरण

### विकल्प 1: स्थैतिक होस्टिंग पर डिप्लॉय करें (नेटलिफ़ाई, वर्सेल, गिटहब पेजेज़)

1. **फ्रंटएंड बनाएं** (यदि पहले से नहीं बनाया गया है):
   ```bash
   cd physical-ai-robotics-textbook/docusaurus
   npm run build
   ```

2. **`build` निर्देशिका को अपने स्थैतिक होस्टिंग प्रदाता पर डिप्लॉय करें**
   - नेटलिफ़ाई के लिए: `build` फ़ोल्डर को खींचें और छोड़ें
   - वर्सेल के लिए: डिप्लॉयमेंट के दौरान `build` फ़ोल्डर का चयन करें
   - गिटहब पेजेज़ के लिए: `build` फ़ोल्डर को `gh-pages` शाखा पर पुश करें

3. **कस्टम डोमेन कॉन्फ़िगर करें** (यदि लागू हो) अपने होस्टिंग प्रदाता में

4. **उचित कैशिंग के लिए कस्टम हेडर सेट करें** (यदि आवश्यक हो):
   ```
   # स्थैतिक संपत्तियों को कैश करें
   /static/* 31536000
   /fonts/* 31536000
   ```

### विकल्प 2: स्व-होस्टेड सर्वर पर डिप्लॉय करें

1. **बिल्ड फ़ाइलें तैयार करें**:
   ```bash
   cd physical-ai-robotics-textbook/docusaurus
   npm run build
   ```

2. **बिल्ड फ़ाइलें अपने वेब सर्वर पर अपलोड करें** (उदाहरण के लिए `/var/www/robotics-textbook` में)

3. **वेब सर्वर कॉन्फ़िगर करें** (उदाहरण नेंगिनएक्स के लिए):
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;

       root /var/www/robotics-textbook;
       index index.html;

       location / {
           try_files $uri $uri/ /index.html;
       }

       # बैकएंड के लिए एपीआई अनुरोधों को प्रॉक्सी करें
       location /api/ {
           proxy_pass http://localhost:8001/;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }
   }
   ```

4. **बैकएंड सर्वर शुरू करें**:
   ```bash
   cd physical-ai-robotics-textbook/backend
   uvicorn main:app --host 0.0.0.0 --port 8001 --workers 4
   ```

### बैकएंड डिप्लॉयमेंट

1. **बैकएंड तैयार करें**:
   ```bash
   cd physical-ai-robotics-textbook/backend
   ```

2. **उत्पादन निर्भरताएँ स्थापित करें**:
   ```bash
   pip install gunicorn
   ```

3. **गुनिकॉर्न के साथ बैकएंड शुरू करें**:
   ```bash
   gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8001
   ```

4. **प्रक्रिया प्रबंधक सेट करें** (लिनक्स पर systemd का उपयोग करके):
   `/etc/systemd/system/robotics-textbook-backend.service` बनाएं:
   ```ini
   [Unit]
   Description=Robotics Textbook Backend
   After=network.target

   [Service]
   User=www-data
   Group=www-data
   WorkingDirectory=/path/to/physical-ai-robotics-textbook/backend
   ExecStart=/path/to/venv/bin/gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8001
   Restart=always

   [Install]
   WantedBy=multi-user.target
   ```

5. **सेवा सक्षम करें और शुरू करें**:
   ```bash
   sudo systemctl enable robotics-textbook-backend
   sudo systemctl start robotics-textbook-backend
   ```

## डेटाबेस सेटअप

### पोस्टग्रेसक्यूएल कॉन्फ़िगरेशन
1. **डेटाबेस बनाएं**:
   ```sql
   CREATE DATABASE robotics_textbook;
   ```

2. **माइग्रेशन चलाएं** (यदि एलेम्बिक का उपयोग कर रहे हैं):
   ```bash
   cd physical-ai-robotics-textbook/backend
   alembic upgrade head
   ```

3. **प्रारंभिक डेटा सेट करें** (यदि आवश्यक हो):
   ```bash
   python scripts/setup_initial_data.py
   ```

## एसएसएल/एचटीटीपीएस कॉन्फ़िगरेशन

1. **एसएसएल प्रमाणपत्र प्राप्त करें** (लेट्स एन्क्रिप्ट का उपयोग करके):
   ```bash
   sudo apt-get install certbot python3-certbot-nginx
   sudo certbot --nginx -d your-domain.com
   ```

2. **एचटीटीपीएस के लिए नेंगिनएक्स कॉन्फ़िगरेशन अपडेट करें**:
   ```nginx
   server {
       listen 443 ssl http2;
       server_name your-domain.com;

       ssl_certificate /path/to/cert.pem;
       ssl_certificate_key /path/to/private.key;

       # ... कॉन्फ़िगरेशन का शेष भाग
   }
   ```

## निरीक्षण और लॉगिंग

### फ्रंटएंड लॉगिंग
- लॉगिंग को उत्पादन में सक्षम करें `REACT_APP_ENABLE_TRANSLATION_LOGGING=true` सेट करके
- अनुवाद लोडिंग त्रुटियों के लिए ब्राउज़र कंसोल की निगरानी करें
- भाषा उपयोग की निगरानी के लिए विश्लेषिकी उपकरणों का उपयोग करें

### बैकएंड लॉगिंग
- `config.py` में लॉगिंग कॉन्फ़िगर करें:
  ```python
  import logging
  logging.basicConfig(level=logging.INFO)
  ```

- बैकएंड लॉग्स की निगरानी करें:
  ```bash
  tail -f /var/log/robotics-textbook-backend.log
  ```

## प्रदर्शन अनुकूलन

### फ्रंटएंड अनुकूलन
1. वेब सर्वर में ज़िप संपीड़न सक्षम करें
2. स्थैतिक संपत्तियों के लिए उचित कैशिंग हेडर सेट करें
3. स्थैतिक संपत्तियों और फ़ॉन्ट के लिए सीडीएन का उपयोग करें
4. अनुवाद फ़ाइलों के लिए आलसी लोडिंग का कार्यान्वयन करें

### बैकएंड अनुकूलन
1. डेटाबेस कनेक्शन के लिए कनेक्शन पूलिंग का उपयोग करें
2. बार-बार एक्सेस किए गए डेटा के लिए कैशिंग कार्यान्वित करें
3. प्रदर्शन के लिए एपीआई एंडपॉइंट्स को अनुकूलित करें
4. सत्र संग्रहण के लिए रेडिस का उपयोग करें (वैकल्पिक)

## समस्या निवारण

### सामान्य समस्याएँ

**समस्या**: भाषा टॉगल दिखाई नहीं दे रहा है
- **कारण**: फ्रंटएंड अनुवाद प्रणाली के साथ बनाया नहीं गया
- **समाधान**: यह सुनिश्चित करें कि लैंग्वेज टॉगल घटक को उचित रूप से एकीकृत किया गया है

**समस्या**: अनुवाद फ़ाइलें लोड नहीं हो रही हैं
- **कारण**: गलत फ़ाइल पथ या अनुमतियाँ
- **समाधान**: यह सत्यापित करें कि अनुवाद फ़ाइलें सही निर्देशिका संरचना में हैं

**समस्या**: प्रमाणीकरण काम नहीं कर रहा है
- **कारण**: बैकएंड एपीआई एंडपॉइंट्स एक्सेस योग्य नहीं हैं
- **समाधान**: बैकएंड सर्वर स्थिति और सीओआरएस कॉन्फ़िगरेशन की जाँच करें

**समस्या**: आरटीएल भाषाएँ सही ढंग से प्रस्तुत नहीं हो रही हैं
- **कारण**: गुम फ़ॉन्ट फ़ाइलें या सीएसएस नियम
- **समाधान**: यह सत्यापित करें कि फ़ॉन्ट फ़ाइलें उचित रूप से होस्ट की गई हैं और सीएसएस में आरटीएल नियम शामिल हैं

### डीबगिंग चरण
1. जावास्क्रिप्ट त्रुटियों के लिए ब्राउज़र कंसोल की जाँच करें
2. ब्राउज़र डेव टूल्स में नेटवर्क अनुरोधों की जाँच करें
3. एपीआई त्रुटियों के लिए बैकएंड लॉग्स की जाँच करें
4. पर्यावरण चर सही ढंग से सेट हैं यह सत्यापित करें
5. सीयूआरएल या पोस्टमैन का उपयोग करके एपीआई एंडपॉइंट्स का परीक्षण करें

## रोलबैक योजना

डिप्लॉयमेंट समस्याओं की स्थिति में:

1. **फ्रंटएंड रोलबैक**: वर्तमान बिल्ड को पिछले संस्करण के साथ बदलें
2. **बैकएंड रोलबैक**: पिछले गिट कमिट पर वापस लौटें और सेवा पुनः आरंभ करें
3. **डेटाबेस रोलबैक**: आवश्यकता पड़ने पर एलेम्बिक का उपयोग करके डाउनग्रेड करें:
   ```bash
   alembic downgrade -1
   ```

## रखरखाव अनुसूची

### दैनिक
- एप्लिकेशन लॉग्स की निगरानी करें
- अनुवाद लोडिंग सफलता दरों की जाँच करें
- प्रमाणीकरण प्रणाली की जाँच करें

### साप्ताहिक
- आवश्यकता पड़ने पर अनुवाद फ़ाइलें अपडेट करें
- प्रदर्शन मेट्रिक्स की समीक्षा करें
- सुरक्षा अद्यतनों की जाँच करें

### मासिक
- निर्भरताओं की समीक्षा और अद्यतन करें
- अनुवाद गुणवत्ता की ऑडिट करें
- एसएसएल प्रमाणपत्र समाप्ति की जाँच करें

---

*यह डिप्लॉयमेंट गाइड सुनिश्चित करता है कि बहु-भाषा अनुवाद प्रणाली सभी सुविधाओं के साथ सही ढंग से डिप्लॉय की गई है। अतिरिक्त सहायता के लिए, डेवलपर दस्तावेज़ीकरण देखें या विकास टीम से संपर्क करें।*
