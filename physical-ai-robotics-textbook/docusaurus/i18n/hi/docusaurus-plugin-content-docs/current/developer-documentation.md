# डेवलपर दस्तावेज़ीकरण: बहु-भाषा अनुवाद प्रणाली

## आर्किटेक्चर अवलोकन

बहु-भाषा अनुवाद प्रणाली को एक रिएक्ट/डॉकुसॉरस एप्लिकेशन के रूप में बनाया गया है जिसमें फास्टएपीआई बैकएंड है। आर्किटेक्चर प्रमाणीकरण एकीकरण और पूर्व-अनुवादित सामग्री वितरण के साथ एक क्लाइंट-सर्वर मॉडल का पालन करता है।

### सिस्टम घटक

```
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│   फ्रंटएंड     │    │   बैकएंड        │    │  अनुवाद        │
│   (डॉकुसॉरस)  │◄──►│   (फास्टएपीआई)  │◄──►│   (जेएसओएन फ़ाइलें) │
│                 │    │                  │    │                 │
│ • भाषा         │    │ • प्रमाणीकरण   │    │ • पूर्व-अनुवादित│
│   कॉन्टेक्स्ट  │    │   सत्र एपीआई    │    │   जेएसओएन फ़ाइलें │
│ • भाषा         │    │ • अनुवाद एपीआई  │    │ • उर्दू और अरबी  │
│   टॉगल        │    │ • उपयोगकर्ता    │    │   सामग्री       │
│ • अनुवाद       │    │   प्राथमिकता    │    │                 │
│   लोडिंग      │    │   एपीआई         │    │                 │
└─────────────────┘    └──────────────────┘    └─────────────────┘
```

### प्रौद्योगिकी स्टैक

- **फ्रंटएंड**: रिएक्ट 18, टाइपस्क्रिप्ट, डॉकुसॉरस 3.x
- **बैकएंड**: पायथन, फास्टएपीआई
- **प्रमाणीकरण**: सत्र-आधारित प्रमाणीकरण
- **फ़ॉन्ट**: नोटो नस्तालिक उर्दू (स्व-होस्टेड), नोटो नस्ख अरबी (गूगल फ़ॉन्ट्स)
- **परीक्षण**: जेस्ट, रिएक्ट टेस्टिंग लाइब्रेरी, एक्स-कोर
- **अनुवाद प्रारूप**: संरक्षित एचटीएमएल संरचना के साथ जेएसओएन

## घटक दस्तावेज़ीकरण

### भाषा कॉन्टेक्स्ट

भाषा कॉन्टेक्स्ट एप्लिकेशन भर में भाषा चयन के लिए वैश्विक स्थिति प्रबंधन प्रदान करता है।

#### इंटरफ़ेस
```typescript
interface LanguageContextType {
  language: 'en' | 'ur' | 'ar' | 'hi';
  setLanguage: (lang: 'en' | 'ur' | 'ar' | 'hi') => void;
  isAuthenticated: boolean;
  loading: boolean;
  error: string | null;
}
```

#### कार्यक्षमता
- वर्तमान भाषा स्थिति का प्रबंधन करता है
- माउंट पर प्रमाणीकरण स्थिति की जाँच करता है
- लोकलस्टोरेज से सहेजी गई भाषा प्राथमिकता लोड करता है
- प्रमाणीकरण-संरक्षित भाषा स्विचिंग को संभालता है
- लोडिंग और त्रुटि स्थितियाँ प्रदान करता है

#### उपयोग
```typescript
import { useLanguage } from '@site/src/context/LanguageContext';

const MyComponent = () => {
  const { language, setLanguage, isAuthenticated } = useLanguage();

  return (
    <div>
      वर्तमान भाषा: {language}
      {isAuthenticated && <button onClick={() => setLanguage('hi')}>हिंदी पर स्विच करें</button>}
    </div>
  );
};
```

### भाषा टॉगल घटक

भाषा टॉगल नेवबार में भाषा चयन के लिए एक ड्रॉपडाउन इंटरफ़ेस प्रदान करता है।

#### प्रॉप्स
- कोई आवश्यक नहीं (भाषा कॉन्टेक्स्ट का उपयोग करता है)

#### विशेषताएँ
- ध्वज इमोजी के साथ वर्तमान भाषा दिखाता है
- सभी समर्थित भाषाओं के साथ ड्रॉपडाउन
- प्रमाणीकरण-जागरूक (लॉग इन नहीं होने पर अक्षम)
- एआरआईए-अनुरूप अभिगम्यता विशेषताएँ
- कीबोर्ड नेविगेशन समर्थन
- आरटीएल-जागरूक स्थिति

#### स्टाइलिंग
- स्कोप्ड शैलियों के लिए सीएसएस मॉड्यूल का उपयोग करता है
- सभी स्क्रीन आकारों के लिए प्रतिक्रियाशील डिज़ाइन
- सुचारु संक्रमण और होवर प्रभाव
- ड्रॉपडाउन स्थिति के लिए उचित आरटीएल समर्थन

### अनुवादित सामग्री घटक

अनुवादित सामग्री घटक अनुवादित सामग्री को लोड करने और प्रदर्शित करने का प्रबंधन करता है।

#### प्रॉप्स
```typescript
interface TranslatedContentProps {
  children: React.ReactNode;
  chapterId: string;  // उदाहरण के लिए, "00-introduction/index"
}
```

#### कार्यक्षमता
- वर्तमान भाषा के आधार पर अनुवाद जेएसओएन गतिशील रूप से लोड करता है
- लोडिंग और त्रुटि स्थितियों का प्रबंधन करता है
- मूल एचटीएमएल संरचना और सीएसएस क्लासों को संरक्षित रखता है
- प्रदर्शन के लिए कैशिंग कार्यान्वित करता है
- अनुवाद विफल होने पर अंग्रेजी सामग्री पर वापस आ जाता है

#### जीवनचक्र
1. जाँच करता है कि क्या वर्तमान भाषा अंग्रेजी है (कोई अनुवाद आवश्यक नहीं)
2. अनुवाद जेएसओएन फ़ाइल लोड करने का प्रयास करता है
3. लाने के दौरान लोडिंग स्थिति प्रदर्शित करता है
4. अनुवादित एचटीएमएल के साथ सामग्री अपडेट करता है
5. अंग्रेजी पर वापस जाने के साथ त्रुटियों का प्रबंधन करता है

## एपीआई दस्तावेज़ीकरण

### प्रमाणीकरण एपीआई

#### GET /api/auth/session
उपयोगकर्ता प्रमाणित है या नहीं यह जाँचें।

**अनुरोध:**
```
GET /api/auth/session HTTP/1.1
Host: localhost:8001
Cookie: session=xxx
```

**प्रतिक्रिया (200 ओके):**
```json
{
  "isAuthenticated": true,
  "user": {
    "id": "user_abc123",
    "email": "user@example.com",
    "name": "John Doe",
    "createdAt": "2025-12-01T00:00:00Z"
  }
}
```

**प्रतिक्रिया (401 अनधिकृत):**
```json
{
  "isAuthenticated": false
}
```

### अनुवाद एपीआई

#### POST /api/translation/translate
ओपनएआई जीपीटी-4 का उपयोग करके सामग्री का अनुवाद करें।

**अनुरोध:**
```
POST /api/translation/translate HTTP/1.1
Host: localhost:8001
Content-Type: application/json
Cookie: session=xxx

{
  "content": "<h1>मूल सामग्री</h1><p>अनुवाद करने के लिए पाठ</p>",
  "source_lang": "en",
  "target_lang": "hi",
  "chapter_id": "00-introduction/index"
}
```

**प्रतिक्रिया (200 ओके):**
```json
{
  "success": true,
  "translated_content": "<h1>अनुवादित सामग्री</h1><p>अनुवादित पाठ</p>",
  "target_lang": "hi",
  "chapter_id": "00-introduction/index",
  "processing_time": 1250
}
```

## अनुवाद फ़ाइल संरचना

### जेएसओएन स्कीमा
```typescript
interface TranslationFile {
  meta: {
    title: string;              // अनुवादित पृष्ठ शीर्षक
    description: string;        // अनुवादित विवरण
    language: 'ur' | 'ar' | 'hi';      // लक्ष्य भाषा
    chapter: string;            // अध्याय पहचानकर्ता
    sourceFile: string;         // मूल .md फ़ाइल पथ
    lastUpdated: string;        // आईएसओ तिथि स्ट्रिंग
    translator: string;         // "ओपनएआई जीपीटी-4" या मैनुअल
    reviewedBy?: string;        // वैकल्पिक समीक्षक
  };

  content: {
    headings: Record<string, string>;     // आईडी → अनुवादित शीर्षक
    paragraphs: Record<string, string>;   // आईडी → अनुवादित पैराग्राफ
    lists: Record<string, string[]>;      // आईडी → अनुवादित सूची वस्तुएँ
    tables?: Record<string, TableData>;   // आईडी → अनुवादित तालिका
    codeComments?: Record<string, string>; // आईडी → अनुवादित टिप्पणियाँ
  };

  html: string;  // पूरा अनुवादित एचटीएमएल (मुख्य सामग्री)
}
```

## आरटीएल कार्यान्वयन

### सीएसएस रणनीति
सिस्टम सीएसएस दिशा विशेषताओं और विशिष्ट आरटीएल शैलियों का उपयोग करके आरटीएल समर्थन कार्यान्वित करता है:

```css
/* आधार आरटीएल शैलियाँ */
[dir='rtl'] {
  text-align: right;
}

/* कोड और आरेखों के लिए एलटीआर संरक्षित रखें */
[dir='rtl'] pre,
[dir='rtl'] code,
[dir='rtl'] .mermaid,
[dir='rtl'] .codeBlockContainer {
  direction: ltr !important;
  text-align: left !important;
}
```

### फ़ॉन्ट लोडिंग
- उर्दू: नोटो नस्तालिक उर्दू (@font-face के साथ स्व-होस्टेड)
- अरबी: नोटो नस्ख अरबी (गूगल फ़ॉन्ट्स के साथ font-display: swap)
- दोनों लोडिंग के दौरान अदृश्य पाठ को रोकने के लिए font-display: swap का उपयोग करते हैं

## प्रदर्शन अनुकूलन

### कैशिंग रणनीति
- ब्राउज़र में अनुवाद जेएसओएन फ़ाइलें कैश की गईं
- लोकलस्टोरेज के साथ भाषा कॉन्टेक्स्ट स्थिति कैश की गई
- प्रमाणीकरण स्थिति 5 मिनट के लिए कैश की गई
- उचित एचटीटीपी हेडर के साथ अनुवाद फ़ाइलें कैश की गईं

### कोड स्प्लिटिंग
- आवश्यकता पर लोड किए गए अनुवाद घटक
- केवल आवश्यकता पर भाषा-विशिष्ट संसाधन लोड किए गए
- ट्री-शेकिंग के साथ बंडल आकार अनुकूलित

## योगदान गाइड

### नए अध्याय जोड़ना

1. **मूल सामग्री बनाएं**: डॉक्स निर्देशिका में उचित स्थान पर अंग्रेजी में अपना नया अध्याय जोड़ें।

2. **अनुवाद निर्देशिकाएँ बनाएं**: प्रत्येक नए अध्याय के लिए, सभी भाषा फ़ोल्डरों में संबंधित निर्देशिकाएँ बनाएं:
   ```
   src/translations/en/new-chapter/
   src/translations/ur/new-chapter/
   src/translations/ar/new-chapter/
   src/translations/hi/new-chapter/
   ```

3. **अनुवाद फ़ाइलें उत्पन्न करें**: स्कीमा संरचना का पालन करते हुए जेएसओएन अनुवाद फ़ाइलें बनाएं। आप अनुवाद स्क्रिप्ट का उपयोग कर सकते हैं या मैन्यल रूप से उत्पन्न कर सकते हैं।

4. **एकीकरण परीक्षण करें**: यदि यह एक मुख्य सामग्री पृष्ठ है तो अपने नए अध्याय को अनुवादित सामग्री घटक के साथ लपेटें।

### नई भाषाएँ जोड़ना

1. **भाषा कॉन्फ़िगरेशन अपडेट करें**: फ्रंटएंड और बैकएंड दोनों में भाषा कॉन्फ़िगरेशन में नई भाषा जोड़ें।

2. **फ़ॉन्ट समर्थन जोड़ें**: नई भाषा के लिए उपयुक्त फ़ॉन्ट फ़ाइलें या सीडीएन लिंक शामिल करें।

3. **आरटीएल समर्थन अपडेट करें**: नई भाषा के लिए उचित आरटीएल/एलटीआर हैंडलिंग सुनिश्चित करें।

4. **अनुवाद फ़ाइलें बनाएं**: नई भाषा में सभी मौजूदा सामग्री के लिए अनुवाद फ़ाइलें उत्पन्न करें।

### विकास कार्यप्रवाह

1. **पर्यावरण सेटअप करें**:
   ```bash
   cd physical-ai-robotics-textbook/docusaurus
   npm install
   ```

2. **विकास सर्वर प्रारंभ करें**:
   ```bash
   npm run start
   ```

3. **बैकएंड एपीआई**: `http://localhost:8001` पर बैकएंड सर्वर चल रहा है यह सुनिश्चित करें

4. **परीक्षण**: परिवर्तन प्रतिबद्ध करने से पहले परीक्षण चलाएं:
   ```bash
   npm test
   ```

### कोड मानक

- प्रकार सुरक्षा के लिए टाइपस्क्रिप्ट का उपयोग करें
- रिएक्ट सर्वोत्तम प्रथाओं का पालन करें
- अभिगम्यता अनुपालन (डब्ल्यूसीएजी 2.1 एए) बनाए रखें
- नई सुविधाओं के लिए व्यापक परीक्षण लिखें
- सार्वजनिक एपीआई और इंटरफ़ेस दस्तावेज़ीकृत करें
- घटक स्टाइलिंग के लिए सीएसएस मॉड्यूल का उपयोग करें
- अनुवादों में मूल एचटीएमएल संरचना और सीएसएस क्लासों को संरक्षित रखें

### सुरक्षा विचार

- सभी अनुवाद एपीआई एंडपॉइंट्स को प्रमाणीकरण की आवश्यकता होती है
- सभी उपयोगकर्ता-प्रदान की गई सामग्री पर इनपुट सत्यापन
- एपीआई एंडपॉइंट्स पर दर सीमांकन
- अनुवाद सामग्री का उचित सैनिटाइज़ेशन
- सुरक्षित सत्र प्रबंधन

## पर्यावरण चर

डॉकुसॉरस निर्देशिका में एक `.env` फ़ाइल बनाएं:
```bash
OPENAI_API_KEY=your_openai_api_key_here
AUTH_API_URL=http://localhost:8001
```

## परीक्षण रणनीति

### इकाई परीक्षण
- भाषा कॉन्टेक्स्ट कार्यक्षमता
- भाषा टॉगल घटक इंटरैक्शन
- अनुवादित सामग्री लोडिंग लॉजिक
- उपयोगिता फ़ंक्शन

### एकीकरण परीक्षण
- प्रमाणीकरण फ्लो एकीकरण
- अनुवाद लोडिंग
- एपीआई संचार
- स्थिति प्रबंधन

### एंड-टू-एंड परीक्षण
- पूर्ण उपयोगकर्ता कार्यप्रवाह
- भाषा स्विचिंग
- सामग्री रेंडरिंग
- अभिगम्यता सुविधाएँ

## डिप्लॉयमेंट विचार

- अनुवाद जेएसओएन फ़ाइलों को स्थैतिक रूप से सेवा किया जाना चाहिए
- फ़ॉन्ट फ़ाइलों को उचित रूप से होस्ट किया जाना चाहिए
- बैकएंड एपीआई एक्सेस योग्य होना चाहिए
- प्रमाणीकरण प्रणाली कार्यात्मक होनी चाहिए
- कैशिंग हेडर उचित रूप से कॉन्फ़िगर किए जाने चाहिए
